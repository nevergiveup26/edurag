智能查询路由 — 三层架构

规则匹配 → 相似度路由 → LLM 推理，逐层递进，精准分发

🔍 用户查询："请帮我用勾股定理解这道题"
1

规则路由

⚡ 0ms · 零成本
命中率 ~60%
关键词 + 正则匹配
问候语 → 直接回复
"批改"/"打分" → 批改模式
"拍照"/"这道题" → 多模态
"查"/"什么是" → 知识库检索
短查询 + 事实类 → 直接检索
2

相似度路由

🕐 ~50ms · 低成本
命中率 ~30%（累计 90%）
向量相似度匹配
将查询向量化后与历史查询库比对
匹配到相似查询 → 复用其策略
使用 embedding 模型做语义匹配
缓存高频查询的路由结果
3

LLM 路由

🧠 ~500ms · 有成本
命中率 100%（兜底）
轻量级 LLM 意图分类
使用 fast-llm（qwen-turbo）做分类
支持 5-8 种策略分类输出
结构化 JSON 输出（含置信度）
复杂/模糊查询的最后保障
Layer 1 →
命中？
→ 否 → Layer 2 →
命中？
→ 否 → Layer 3 →
分类
📖

直接检索

直接向量检索知识库
适用简单事实查询

💡

HyDE 检索

生成假设文档再检索
适用概念性问题

🔀

子查询分解

拆分子问题分别检索
适用复合/多步问题

🔄

回溯检索

多轮检索+质量评估
适用需要深度搜索的问题
